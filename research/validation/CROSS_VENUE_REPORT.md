# 교차 거래소 펀딩레이트 검증 리포트 (CROSS_VENUE_REPORT)

- 생성: 2026-07-16 08:08 UTC / 스크립트: `research/validation/cross_venue_funding.py`
- 목적: 펀딩 신호가 Binance 고유 산출물이 아니라 시장 공통 현상인지 3사(Bybit·OKX, 참고 Bitget) 교차검증
- 대상: universe.json 상위 12개 심볼 (ETHUSDT, BTCUSDT, SOLUSDT, XRPUSDT, DOGEUSDT, ADAUSDT, WLDUSDT, NEARUSDT, ONDOUSDT, SUIUSDT, LINKUSDT, BCHUSDT)
- 수집 구간 목표: 2024-01-01 ~ 현재 (UTC, 8h 펀딩 이벤트). 실제 확보 구간은 거래소별 API 한도에 따라 다름(아래 커버리지 제약 참조).

## 방법

- 각 거래소 펀딩 이벤트를 UTC 정시 8h 버킷(00/08/16시)으로 반올림 정렬하고 버킷 내 합산. 4h/1h 주기 심볼도 합산 누적이라 8h 거래소와 비교 가능. (반올림을 쓰는 이유: Binance 캐시 자체에 수 ms 정산 지연 타임스탬프가 섞여 있어 내림 정렬 시 오분류 위험이 있음.)
- 7d APR 스코어 = 직전 21버킷(7일) 펀딩 합 x 365/7 (= 기존 전략 코드의 rolling mean x 3 x 365와 동일값). 결측 버킷이 낀 창은 제외(전략 신호와 동일한 연속성 요건).
- 진입신호: 7d APR > 15% (W2c/F1f 임계). 진입일치(활성)은 한쪽이라도 신호가 켜진 버킷만 대상.
- 구간 분해: Binance 스코어 기준 저펀딩 |7d APR| < 5%, 고펀딩 |7d APR| > 15%.
- 판정 기준(주 판정만 적용): Binance vs Bybit 고펀딩 구간 부호 일치율 >= 90% (집계 N >= 30)이면 "펀딩 = 시장 공통, 거래소 재현성 차단기 해제 근거". OKX·Bitget은 단기/제한 커버리지라 보조 확인으로만 사용하고 판정을 좌우하지 않음.

## 커버리지 제약

- Bybit: 2024-01-01부터 현재까지 endTime 워크백으로 전 구간 수집 (장기, 주 판정 대상).
- OKX: 공개 funding-rate-history 엔드포인트가 최근 약 3개월만 반환(실측: after 90일 전 데이터 있음, 100일 전 빈 응답). after 워크백이 자연스럽게 그 지점에서 빈 페이지를 받고 종료되므로 2024-01-01 목표는 도달 불가 -> 단기 보조 확인으로 강등. 표의 기간(일) 열이 실제 겹침 구간.
- Bitget: wave1 캐시(BTC/ETH/SOL만, 약 133일). 그 외 심볼은 N/A. 보조 확인/참고용.
