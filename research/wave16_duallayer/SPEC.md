# Wave-16 사전등록 — 이중수익 캐리: 펀딩 + 현물 대여이자 (동결: 2026-07-22)

## 발견 (실측, 2026-07-22)
wave-15 B1은 현물 레그 추가수익을 **가정 2%**로 뒀고, 그래서 승격을 기각했다. 실제로 재보니 가정이 **너무 낮은 쪽으로 틀렸다**:
- OKX 공개 `lending-rate-summary` 170종: **중앙값 10.31%**, 91종이 10% 초과, 알트 최고 22~69%. 메이저는 낮음(BTC 0.50%, ETH 1.50%, SOL 4.65%).
- **현재 펀딩은 전 코인 11% APR로 균일**(0.01%/8h 하한) → 진입임계 15% 미달 → L4 대기 중. 반면 대여이자는 1~22.5%로 분산이 크다.
- 합산 시 THETA 33.5%, GALA 26.2%, AVAX 22.0% — **저펀딩 레짐에서 대여이자가 유일한 차별 요인이자 임계 돌파 수단**.
- Bitget 대출 이자(`earn/loan/public/coinInfos`)도 공개됨(USDT 7일 2.56%) — 실행 거래소 측 데이터 존재 확인.

**가설: 캐리 랭킹을 펀딩 단독이 아니라 (펀딩 + 대여이자)로 하면, 특히 저펀딩 레짐에서 수익이 구조적으로 개선된다.**

## 치명적 한계 (먼저 명시 — 이 wave의 결론을 제한한다)
1. **대여이자 과거 시계열이 공개되지 않는다.** 현재 스냅샷만 존재 → **시계열 백테스트 불가**. 이 wave는 "과거에 얼마 벌었나"가 아니라 **"현재 단면에서 구조가 성립하나"**를 검정한다. 이 구분을 리포트 제목·결론에 명시할 것.
2. **대여 가능성 미검증**: 헤지 중인 현물을 대여하면 즉시 회수 가능한지(청산 시 recall 지연), 한도·계층 존재 여부, 대여이자가 대여자 수취분인지 차입자 지불분인지(플랫폼 스프레드) 전부 미확인. **운영 리스크로 명시하고 수익에서 보수적 할인**.
3. **거래소 분리**: OKX 대여 vs Bitget 실행이면 자본이 나뉜다(wave-14 M6/M7에서 확인한 거래소 신용리스크 재발). Bitget 자체 대여 가능 여부를 우선 확인하고, 불가 시 이 구조는 거래소간 노출로 등급 하향.

## 후보 5개 (동결, 사후 추가 금지)
공통: $100/활성 $90/레그 $45, 델타중립 1x, wave-13 실측비용, top200 유니버스(L4 승계)
| ID | 정의 |
|---|---|
| E0 | **L4 재현(대여 없음) = 기준선** — 동일 단면·동일 비용으로 재산출 |
| E1 | 랭킹 = 펀딩 APR만(=L4), 진입 후 현물 레그를 **실측 대여이자**로 운용(수익 가산) |
| E2 | **랭킹 = 펀딩 + 대여이자 합산**, 임계 15% 합산 기준 — 핵심 가설 |
| E3 | E2 + 대여이자를 **50% 할인** 적용(플랫폼 스프레드·미체결 보수 가정) |
| E4 | E2 + 대여이자 **0% 가정**(대여 실패 시나리오) — 랭킹만 합산, 수익은 펀딩만 |

E3/E4는 스트레스다. **E4가 E0보다 나쁘면 "합산 랭킹 자체가 해롭다"는 뜻**이므로 반드시 산출.

## 방법 (시계열 불가 → 단면 분석 + 시나리오)
1. 현재 단면: 전 심볼의 (펀딩 APR, 대여이자, 유동성, 실측 슬리피지) 수집·저장.
2. 각 후보의 **현 시점 선택 종목과 기대 연환산**을 산출(포트폴리오 구성 시뮬).
3. **과거 검증 대체물**: 대여이자를 과거 구간에 적용할 수 없으므로, 대신 **현재 대여이자를 상수로 두고 과거 펀딩 시계열에 얹은 하한 추정치**를 별도 표기(낙관 아님을 명시 — 대여이자는 변동하며 고펀딩기엔 더 높았을 가능성이 크나 확인 불가).
4. 게이트: 시계열 MC/블록셔플은 펀딩 부분에만 적용 가능 → **대여이자 부분은 게이트 미적용**임을 표기하고, 승격은 "구조 성립 + 스트레스(E3/E4) 생존"으로만 판정. **"검증된 수익률"로 표기 금지.**

## 판정
- E2가 E0 대비 개선 ∧ E3(50% 할인) 개선 유지 ∧ E4(대여 0%) ≥ E0 → **구조 유효**로 등급 부여(단 "단면 근거, 시계열 미검증" 라벨 필수).
- E4 < E0면 합산 랭킹 기각.
- 운영 검증 항목(대여 회수 가능성·한도·수취율)을 **미해결 목록**으로 리포트에 명시하고, 실자금 적용 전 확인 필요 항목으로 남길 것.
